# HMI Host 架构说明

## 1. 目标

本项目是基于 Flutter 的跨平台上位机，用于通过双串口与 STM32 打包机固件交互。

## 2. 模块分层

```text
UI Layer
  lib/features/hmi/hmi_dashboard_page.dart
    - 页面布局、参数输入、USART1 HMI Session 日志展示
    - 用户操作转发到 Controller（旧业务协议入口已移除）

Application Layer
  lib/features/hmi/hmi_controller.dart
    - 串口 A/B 连接管理
    - 端口 A: 打包机 20B 指令发送封装与响应解析（0x40~0x4C）
    - 端口 B: `0x7E` 单帧收发与 HMIS-BAM 分片收发，处理 HMI Session 参数、日志和快捷控制

Protocol Layer
  lib/core/protocol/hmi_frame.dart
  lib/features/hmi/hmi_hmis_bam.dart
  lib/features/hmi/hmi_session_single_frame.dart
  lib/features/hmi/hmi_protocol.dart
    - 20 字节固定帧编解码
    - USART1 `FUNC=0x7E` Session 单帧编解码
    - USART1 `FUNC=0x7F` HMIS-BAM 分片、重组、ACK/NACK
    - 打包机现行功能码语义与结果码解码

Transport Layer
  lib/core/serial/serial_transport.dart
  lib/core/serial/serial_transport_impl.dart
    - 串口枚举、打开、读写、关闭
  lib/core/serial/android_usb_serial_transport.dart
    - Android USB Host OTG 串口实现（MethodChannel/EventChannel）
  lib/core/serial/desktop_serial_transport.dart
    - 桌面 `flutter_libserialport` 实现
```

## 3. 关键设计决策

- 使用 `SerialTransport` 抽象隔离平台实现，避免业务层耦合插件细节。
- Android 串口改为原生 USB Host（`usb-serial-for-android`），桌面平台继续使用 `flutter_libserialport`。
- 统一在 `HmiController` 中做字节流缓冲与拆包，避免 UI 处理协议细节。
- `HmiFrame` 对 payload 固定 16 字节，保证帧长度稳定。
- USART1 日志不再按原始 ASCII 文本处理，统一按 `0x7E` 或 HMIS-BAM 重组后的 HMI Session `LOG_PUSH` 解析。
- 端口职责固定：`USART3/端口 A` 主要跑 20 字节主协议，`USART1/端口 B` 跑 20 字节 HMI Session 单帧/BAM；
  默认发送路由与接收解析都不再跨端口自动回退。
- HMI Session `0x55 0xAA` 仅作为 BAM 内部逻辑帧，不直接上 USART1 物理层；`0x7E` 单帧直接承载 `TYPE/SEQ/CMD/FLAGS/LEN/PAYLOAD(10)`；节点地址由界面输入控制，不写死为 `0xFA`。

## 4. 扩展建议

- 新增功能码时先扩展 `HmiController` 的发送 API，再补 UI 入口。
- 若需支持自动重连，可在 `HmiController` 增加重连状态机。
- 若需复杂业务编排，建议新增 `service` 层，不要把流程写入 UI。
