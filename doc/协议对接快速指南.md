# 打包机 HMI 协议对接快速指南

## 1. 当前有效链路

- 左链路（端口 A）：`USART3 / 9600 / 8N1`
- 协议：固定 `20` 字节帧 + `CRC16-Modbus`（低字节在前）
- 功能码：主协议 `0x40~0x4D`，同时可接收 HMI Session `0x7E/0x7F`
- HMI 口径：只发送/接收 `20` 字节主协议，不解析 HMIS-BAM 或 HMI Session

- 右链路（端口 B）：`USART1 / 9600 / 8N1`
- 协议：固定 `20` 字节 HMI Session 承载，短帧 `FUNC=0x7E`，长帧/推送 `FUNC=0x7F`
- 用途：参数调试 + 系统信息读取 + 日志显示
- HMI 口径：单参数读/写等短交互优先走 `0x7E`；全量参数同步、目录、设备信息、日志/事件/栈快照等长响应或推送走 `0x7F` BAM；`0x55 0xAA` HMI Session 只在 BAM 重组后作为内部逻辑帧处理；不解析 `0x40~0x4D` 主协议
- 当前 BAM 口径：V3 `uint32_t TID`，`Stop-and-Wait` 逐片 ACK/NACK，单片有效载荷 `8B`，静态上限 `512B / 64片`

## 2. 打包机主协议（20B）

> **完整协议规范请见固件端主文档：** [`打包机上位主控系统说明书.md`](/root/develop/Packaging_machine_V1.0/doc/打包机上位主控系统说明书.md)
> 包含：帧格式、功能码语义、响应字段、状态快照、报警码定义。
>
> HMI 侧仅维护此处速查摘要，详细字段以固件端文档为准。

端口 A 主协议帧格式速查（20B 固定帧 / CRC16-Modbus / 地址 `0xFA`）：

| 字节 | 含义 |
|---|---|
| 0 | 地址 |
| 1 | 功能码（`0x40~0x4C`） |
| 2~17 | 数据域（16字节） |
| 18~19 | CRC16-Modbus（L/H） |

## 3. USART1 HMI Session 参数与日志

`FUNC=0x7E` 单帧格式：

| 字节 | 含义 |
|---|---|
| 0 | 地址（界面可编辑，默认 `0xFA`） |
| 1 | 功能码（固定 `0x7E`） |
| 2 | `TYPE` |
| 3~4 | `SEQ`，低字节在前 |
| 5 | `CMD` |
| 6 | `FLAGS` |
| 7 | `LEN`，最大 `10` |
| 8~17 | `PAYLOAD[10]`，不足补 `0x00` |
| 18~19 | CRC16-Modbus（L/H） |

适用口径：

- 单个参数读取：`GET_PARAM_VALUES_BATCH(0x12)`，请求 payload 为 `[1, param_id]`，响应 payload 为 `[1, param_id, 0, value_le32]`，走 `0x7E`
- 单个参数写入：`SET_PARAM_VALUES_BATCH(0x13)`，请求 payload 为 `[1, param_id, 0, value_le32]`，响应 payload 不超过 `10B`，走 `0x7E`
- 快捷控制、点动、订阅开关等 payload `<=10B` 的短命令优先走 `0x7E`

HMIS-BAM 外层帧格式：

| 字节 | 含义 |
|---|---|
| 0 | 地址（界面可编辑，默认 `0xFA`） |
| 1 | 功能码（固定 `0x7F`） |
| 2~5 | TID，低字节在前，`0` 非法 |
| 6 | 分片序号，`0xFE` ACK，`0xFF` NACK |
| 7 | 分片总数 |
| 8 | 本片 Session 字节数 |
| 9~16 | HMI Session 原始帧片段（8 字节） |
| 17 | `RESERVED`，固定 `0x00`，参与 CRC |
| 18~19 | CRC16-Modbus（L/H） |

控制帧口径：

- `FRAG_INDEX=0xFE`：ACK
- `FRAG_INDEX=0xFF`：NACK
- `TID`：ACK/NACK 绑定的事务
- `PAYLOAD[0]`（偏移 9）：针对的 `FRAG_INDEX`
- `PAYLOAD[1]`（偏移 10）：状态码
- `PAYLOAD[2]`（偏移 11）：`NEXT_EXPECTED_INDEX`
- 控制帧同样要求偏移 17 的 `RESERVED=0x00`

- 参数读写：使用 `GET_PARAM_VALUES_BATCH(0x12)` / `SET_PARAM_VALUES_BATCH(0x13)`
- 全量参数同步：HMI 使用批量 `GET_PARAM_VALUES_BATCH(0x12)`，强制走 `0x7F` BAM
- 参数保存/加载/默认值：使用 `0x14 / 0x15 / 0x16`
- 系统信息：使用 `GET_DEVICE_STATUS(0x20)`
- 报警读取：使用 `GET_ALARM_STATUS(0x21)`
- 日志推送：`TYPE=0x04 / CMD=0x41`，payload=`[level][text...]`

日志显示口径：

- 固件将日志封装为内部 Session `LOG_PUSH`；短推送可走 `0x7E`，带文字的日志/事件/栈快照通常走 USART1 HMIS-BAM 分片发送
- HMI 解析 `0x7E` 单帧或 BAM 重组后的 Session 日志帧，不依赖原始 ASCII 文本行或裸 `55 AA` 长帧
- Android 自动日志落盘写入应用私有目录；手动导出时由用户选择目标位置，HMI 将 `protocol_logs.txt`、原始滚动日志 `jsonl`、栈统计 `jsonl` 一并打包为 `zip`
- 栈水位相关新口径：
  - `STACK_LEVEL=<min>`：用于 HMI 提取单值样本
  - `STACK_SNAPSHOT_BEGIN`
  - `STACK_TASK NAME=<TaskName> TOTAL=<words> FREE=<words>`
  - `STACK_SNAPSHOT_END`
- 当前任务快照口径：
  - 任务集合固定为 `ProtoTask / StateMachineTask / MotorTask / AdcTask / CommTask / MonitorTask / HeaterTask`
  - `TOTAL` / `FREE` 单位均为 FreeRTOS 栈字数（word，STM32F1 当前为 4 字节/word）
  - HMI 仅在收齐一整份快照后刷新任务表格与结构化栈统计 `jsonl`

系统信息读取口径：

- 固件不再周期性主动推送状态
- HMI 需要按需发送 `GET_DEVICE_STATUS(0x20)` 获取系统信息

## 4. 门禁规则

USART1 HMI Session 参数读写/保存/加载仅在以下条件满足时允许：

- 状态机 `IDLE`
- 运行许可为 `STOP`
- 启动序列已完成

否则固件返回 `REJECTED_STATE`，不执行业务读写。
