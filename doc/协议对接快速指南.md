# UART3 协议对接快速指南

本指南是对 `下位机系统说明书.md` 的上位机落地提炼，重点给出 Flutter 端需要的最小集合。

## 1. 基本参数

- 接口: UART3
- 推荐波特率: `115200`
- 数据格式: `8N1`
- 帧长: 固定 `20` 字节
- 地址:
  - 请求: `0xAF`
  - 响应: `0xBF`
- 校验: `CRC16-Modbus`（低字节在前）

## 2. 帧格式

| 字节范围 | 字段 | 说明 |
|---|---|---|
| byte0 | addr | 地址字节 |
| byte1 | func | 功能码 |
| byte2~17 | data[16] | 负载，未使用字节补 `0x00` |
| byte18~19 | crc | CRC16，低字节在前 |

## 3. 已接入功能码

| 功能码 | 名称 | HMI 控制器方法 |
|---|---|---|
| `0x10` | 初始化查询 | `sendInitQuery()` |
| `0x09` | 状态查询 | `sendStatusQuery()` |
| `0x07` | 取货开锁 | `sendUnlock()` |
| `0x01` | 订单下发 | `sendOrder()` |

## 4. 上位机对接注意点

- 串口字节流可能出现粘包与拆包，不能假设一次 read 就是一帧。
- 收到非法 CRC 帧时应丢弃并继续向后同步帧头。
- 上位机发送参数应限制在 `0..255`，避免超出单字节范围。

## 5. 测试建议

1. 先测 `0x10`（确认设备在线）
2. 再测 `0x09`（确认反馈链路）
3. 再测 `0x07/0x01`（确认业务动作链）
