# 打包机 HMI 协议对接快速指南

## 1. 当前有效链路

- 左链路（端口 A）：`USART3 / 9600 / 8N1`
- 协议：固定 `20` 字节帧 + `CRC16-Modbus`（低字节在前）
- 功能码：仅保留 `0x40~0x4C`
- HMI 口径：只发送/接收 `20` 字节主协议，不解析 USART1 Session

- 右链路（端口 B）：`USART1 / 9600 / 8N1`
- 协议：HMI Session，帧头 `0x55 0xAA`
- 用途：参数调试 + 系统信息读取 + 日志显示
- HMI 口径：只发送/接收 USART1 Session，不解析 `20` 字节主协议

## 2. 打包机主协议（20B）

帧格式：

| 字节 | 含义 |
|---|---|
| 0 | 地址（常用 `0xFA`） |
| 1 | 功能码（`0x40~0x4C`） |
| 2~17 | 数据域（16字节） |
| 18~19 | CRC16-Modbus（L/H） |

## 3. USART1 Session 参数与日志

- 参数读写：使用 `GET_PARAM_VALUES_BATCH(0x12)` / `SET_PARAM_VALUES_BATCH(0x13)`
- 参数保存/加载/默认值：使用 `0x14 / 0x15 / 0x16`
- 系统信息：使用 `GET_DEVICE_STATUS(0x20)`
- 报警读取：使用 `GET_ALARM_STATUS(0x21)`
- 日志推送：`TYPE=0x04 / CMD=0x41`，payload=`[level][text...]`

日志显示口径：

- 固件将日志封装为 USART1 Session `LOG_PUSH`
- HMI 仅解析 Session 日志帧，不依赖原始 ASCII 文本行
- Android 自动日志落盘写入应用私有目录；手动导出时由用户选择目标位置，HMI 将 `protocol_logs.txt`、原始滚动日志 `jsonl`、栈统计 `jsonl` 一并打包为 `zip`
- 栈水位相关新口径：
  - `STACK_LEVEL=<min>`：用于 HMI 提取单值样本
  - `STACK_SNAPSHOT_BEGIN`
  - `STACK_TASK NAME=<TaskName> TOTAL=<words> FREE=<words>`
  - `STACK_SNAPSHOT_END`
- 当前任务快照口径：
  - 任务集合固定为 `ProtoTask / StateMachineTask / MotorTask / AdcTask / CommTask / MonitorTask / HeaterTask`
  - `TOTAL` / `FREE` 单位均为 FreeRTOS 栈字数（word，STM32F1 当前为 4 字节/word）
  - HMI 仅在收齐一整份快照后刷新任务表格与结构化栈统计 `jsonl`

系统信息读取口径：

- 固件不再周期性主动推送状态
- HMI 需要按需发送 `GET_DEVICE_STATUS(0x20)` 获取系统信息

## 4. 门禁规则

USART1 Session 参数读写/保存/加载仅在以下条件满足时允许：

- 状态机 `IDLE`
- 运行许可为 `STOP`
- 启动序列已完成

否则固件返回 `REJECTED_STATE`，不执行业务读写。
