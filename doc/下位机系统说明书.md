# STM32_XYZ 自动出货打包机系统说明文档

## 一、系统概述

本系统是一套基于 **STM32F103RC** 微控制器的自动出货打包机控制系统，采用 **FreeRTOS** 实时操作系统，用于精密的多轴运动控制、传感器采集、通讯交互及自动化流程管理。

### 主要功能

| 功能模块 | 说明 |
|----------|------|
| 多轴运动控制 | X/Y/Z/IR四轴步进电机(TIM4 PWM)，独立控制，支持寻零 |
| 传送带驱动 | TIM1互补PWM(IR2104 H桥) + TIM5 CH1(出货电机)，支持软启动 |
| 传感器采集 | 19路数字输入(IN1~IN19)，支持消抖和边沿检测 |
| 多协议通讯 | UART1(打包模块) / UART2(M109货柜/储物板) / UART3(APP上位机) |
| 状态管理 | 8态状态机(dev_state_e)，防止业务流程重入 |
| LED指示 | 单LED，3种模式(空闲1s闪/正常接收500ms/错误200ms快闪) |
| CRC校验 | CRC16-Modbus算法，全部通讯帧校验 |

### MCU资源

| 资源 | 规格 |
|------|------|
| 芯片型号 | STM32F103RC |
| FLASH | 256KB |
| SRAM | 48KB |
| 主频 | 72MHz |
| 定时器使用 | TIM1(NMOS H桥), TIM4(步进电机PWM), TIM5(出货电机), TIM8(通用计时) |
| UART | USART1(打包), USART2(M109/储物板), USART3(APP上位机) |

### 2. FreeRTOS配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 总堆大小 | 16KB | FreeRTOS动态内存池 |
| 系统节拍 | 1000Hz | 1ms时基 |
| 最大优先级 | 56级 | 支持多优先级调度 |
| 最小栈大小 | 128字 (512B) | 空闲任务栈 |

### 3. 任务配置

| 任务名称 | 功能 | 栈大小 | CMSIS优先级 | 类型 |
|----------|------|--------|------------|------|
| INITTask | 上电寻零(X→Y→Z) | 1024B (256×4) | BelowNormal | 一次性(完成后自删除) |
| MotionControlTa | 运动控制/指令执行 | 1024B (256×4) | High | 常驻 |
| BusinessTask | 业务流程/协调/调度 | 1024B (256×4) | AboveNormal | 常驻 |
| UART3CommTask | APP上位机协议解析 | 1024B (256×4) | High | 常驻 |
| UART2CommTask | M109/储物板帧接收 | 1024B (256×4) | High | 常驻 |
| UART1Comm | 打包模块透传 | 768B (192×4) | AboveNormal | 常驻 |
| StepperTestTask | 步进电机测试 | 1024B (256×4) | Normal | 常驻 |

**任务总栈占用**: 768+1024×4+768+1024 = 6656字节

### 4. 消息队列

| 队列名称 | 深度 | 消息大小 | 用途 |
|----------|------|----------|------|
| xQueue_Motion | 8 | 32字节 (motion_msg_t) | 业务→运动控制: 运动指令+参数 |
| xQueue_CmdApp | 8 | 64字节 (cmd_msg_t) | 串口3(APP)→业务: APP端协议指令 |
| xQueue_CmdM109 | 16 | 64字节 (cmd_msg_t) | 串口2(M109)→业务: M109响应帧 |
| xQueue_CmdLocker | 16 | 64字节 (cmd_msg_t) | 串口2(储物板)→业务: 储物板响应帧 |

> 注: UART2共享总线上的M109与储物板(Locker)响应分别路由到独立队列, 通过 src_port=2 标识来源, 避免互相消费。队列溢出通过 `g_queue_overflow_m109` / `g_queue_overflow_locker` 全局计数器记录。

### 5. 同步机制

| 类型 | 名称 | 用途 |
|------|------|------|
| 事件组 | xEvent_Motion | 运动完成(bit0)/失败(bit1)/超时(bit2)/传感器触发(bit3)/初始化完成(bit4)/初始化失败(bit5) |
| 二值信号量 | xSem_UART1 | 串口1中断→串口1任务通知 |
| 二值信号量 | xSem_UART2 | 串口2中断→串口2任务通知 |
| 二值信号量 | xSem_UART3 | 串口3中断→串口3任务通知 |
| 软件定时器 | xTimer_LED | 10ms周期, 驱动LED状态机闪烁 |

### 6. 内存估算

```
FreeRTOS堆总量:         16KB (16384 字节)
├── 任务栈:             6.5KB (6656 字节)
├── 队列内存:           ~0.8KB (32×8 + 64×8 = 768 字节)
├── 事件组+信号量×3:    ~0.3KB (300 字节)
├── 软件定时器:         ~0.1KB (100 字节)
└── 剩余可用:           ~8.3KB (8560 字节)
```

> 注意: 实际剩余内存可通过 `xPortGetFreeHeapSize()` 运行时获取。

### 2. 指令处理流程

| 功能码 | 指令名称 | 处理路径 | 说明 |
|--------|----------|----------|------|
| 0x01 | 订单下发 | UART3→Biz→M109(UART2)→NMOS→传感器→UART3反馈 | 核心出货流程 |
| 0x03 | 打包封口 | UART3→Biz→UART1(打包)→IR反转→UART3反馈 | IR限时运动 |
| 0x05 | 储物格存放 | UART3→Biz→开门(UART2)→多轴运动→UART3反馈 | 含Y/X/Z/归零 |
| 0x07 | 取货开锁 | UART3→Biz→开门(UART2)→UART3反馈 | 简化等待 |
| 0x09 | 状态查询 | UART3→Biz→UART2查询储物板→UART3响应 | 锁状态+占用状态 |
| 0x0B | 设备测试 | UART3→Biz→对应运动→UART3反馈 | 货道/升降/门锁测试 |
| 0x0C | 退货 | UART3→Biz→开锁→等待→UART3反馈 | 开锁并等待放回 |
| 0x10 | 初始化查询 | UART3→Biz→检查STATE_READY→UART3反馈 | 仅就绪后响应一次 |

### 3. 全局状态管理

使用枚举类型 `dev_state_e` 管理设备状态，通过函数接口读写（支持加锁扩展）。

```
状态流转图:
  IDLE → INITIALIZING → READY ⇄ DELIVERING/PACKING/STORING/PICKING
                          ↑                    ↓
                          └─── 完成/超时 ──────┘
                       ERROR (任意状态均可进入)
```

| 状态 | 枚举值 | 含义 |
|------|--------|------|
| STATE_IDLE | 0 | 空闲(上电初始) |
| STATE_INITIALIZING | 1 | 正在初始化(寻零中) |
| STATE_READY | 2 | 就绪(可接收指令) |
| STATE_DELIVERING | 3 | 出货中 |
| STATE_PACKING | 4 | 打包中 |
| STATE_STORING | 5 | 存储中 |
| STATE_PICKING | 6 | 取货中 |
| STATE_ERROR | 7 | 错误(需复位) |

**接口函数**:
- `App_State_Init()` — 初始化为IDLE
- `App_State_Set(state)` — 设置状态
- `App_State_Get()` — 获取当前状态
- `App_State_GetName()` — 获取状态字符串名
- `App_State_IsReady()` — 检查是否可接收新指令

### 2. 输出接口

#### 2.1 步进电机驱动 (4轴, TIM4 PWM)

| 轴 | 脉冲(PUL) | 方向(DIR) | 使能(EN) | TIM4通道 | 功能 |
|----|-----------|----------|---------|---------|------|
| X(ST1) | PB6 | PB4 | PB5 | CH1 | 横向传送 |
| Y(ST2) | PB7 | PD2 | PB3 | CH2 | 纵向推板 |
| Z(ST3) | PB8 | PC11 | PC12 | CH3 | 垂直升降 |
| IR(ST4) | PB9 | PA15 | PC10 | CH4 | 反转打包 |

**使能逻辑**: EN低电平=使能, EN高电平=失能
**速度控制**: 频率 = 72MHz / (PSC+1) / (ARR+1), PSC=0, 通过调整ARR改变频率
**频率范围**: 100Hz ~ 100kHz
**占空比**: 固定50% (CCR = ARR/2)
**寻零频率**: 2000Hz (CFG_SEEKZERO_FREQ)
**运行频率**: X/Z=8000Hz, Y=10000Hz (CFG_STEPPER_X/Y/Z_RUN_FREQ)
**寻零超时**: 30秒 (BSP_STEPPER_SEEKZERO_TIMEOUT_MS)
**Y轴运动超时**: 25秒 (BSP_STEPPER_Y_TIMEOUT_MS, 行程较长)
**Y轴使能策略**: Y轴仅在零点(IN5)时允许失能，非零点位置必须保持使能防止丢步/滑落。handle_stop_all()仅失能X/Z轴，不失能Y轴。
**缓启动**: 每次启动从1100Hz逐级加速到目标频率。默认10级×20ms=200ms, Z轴专用20级×20ms=400ms(加速更平缓, 防止失步)。加速期间持续检测传感器，到位立即停止。起始频率不可低于1100Hz(TIM4为16位定时器, ARR≤65535, 72MHz/65536≈1099Hz)。

#### 2.2 NMOS电机驱动

**电机1 — TIM1互补PWM (IR2104 H桥)**:

| 引脚 | 功能 | 说明 |
|------|------|------|
| PA8 | TIM1_CH1 | PWM上桥臂驱动(HIN) |
| PB13 | TIM1_CH1N | PWM下桥臂驱动(LIN, 互补) |
| PC9 | IR2104_1SD | 使能(高有效), 低电平=刹车 |

**电机2 — TIM5 CH1 (传送带1, 出货传送带)**:
- 用于出货流程中的传送带电机正转控制
- 函数: `BSP_NMOS_TIM5_CH1_RampStart(duty, step_ms)`

**电机3 — TIM5 CH2 (传送带2, 转盘传送带)**:
- 用于货柜号≥4时的转盘传送带电机控制
- IN17(PC15)检测到位后停止
- 函数: `BSP_NMOS_TIM5_CH2_RampStart(duty, step_ms)`

**共用PWM特性**: 
- 死区: 内置(防止上下臂同导)
- 最大安全占空比: 96% (BSP_NMOS_MAX_SAFE_DUTY=960)
- 软启动: 每次递增10‰ (BSP_NMOS_RAMP_STEP=10)
- 目标占空比: 70% (CFG_NMOS_TARGET_DUTY=700)

#### 2.3 LED指示灯

| 模式 | 周期 | 说明 |
|------|------|------|
| LED_MODE_IDLE | 1000ms | 上电默认, 慢闪 |
| LED_MODE_NORMAL_RX | 500ms闪一次 | 串口正常收帧后触发 |
| LED_MODE_ERROR | 200ms快闪5次 | 错误上报后触发 |

- 通过10ms软件定时器(xTimer_LED)驱动BSP_LED_Process()状态机
- 闪烁完成后自动恢复IDLE模式

## 五、通讯协议

### 1. UART3协议 (APP上位机通讯, 主协议)

**帧格式** (20字节, 固定长度):
```
┌──────────┬──────────┬──────────────────┬──────────┐
│ 地址码   │  指令码  │    数据域        │  CRC16   │
│ 1B       │  1B      │    16B           │  2B(LE)  │
├──────────┼──────────┼──────────────────┼──────────┤
│ [0]      │ [1]      │ [2] ~ [17]       │[18][19]  │
└──────────┴──────────┴──────────────────┴──────────┘
```

**地址码定义**:
- `0xAF` — APP→设备 (下行指令)
- `0xBF` — 设备→APP (上行反馈)

**功能码完整列表**:

| 功能码 | 方向 | 名称 | 数据域说明 |
|--------|------|------|-----------|
| 0x01 | APP→设备 | 订单下发 | [0]订单号 [1]数量 [2]货柜地址 [3]货层 [4]货道 |
| 0x02 | 设备→APP | 投递状态反馈 | [0]订单号 [1]结果 [2]失败原因 |
| 0x03 | APP→设备 | 打包封口 | [0]订单号 [1]封口指令(0x01) |
| 0x04 | 设备→APP | 封口完成 | [0]封口状态(0xAA=完成) |
| 0x05 | APP→设备 | 储物格存放 | [0]打包编号 [1]储物格号 |
| 0x06 | 设备→APP | 存放反馈 | [0]订单号 [1]结果 |
| 0x07 | APP→设备 | 取货开锁 | [0]订单号 [1]储物格号 |
| 0x08 | 设备→APP | 开锁状态反馈 | [0-1]格号 [2]开锁结果 [3]货物状态 |
| 0x09 | APP→设备 | 状态查询 | [0]查询类型(0x01=储物格状态) |
| 0x09 | 设备→APP | 状态查询响应 | [0-1]锁状态(high,low) [2-3]占用状态(high,low) |
| 0x0A | 设备→APP | 错误上报 | [0]错误码(参见§10统一错误码表, 0x01~0x2D) [1-15]预留=0x00 |
| 0x0B | 双向 | 设备测试 | [0]类型 [1-2]目标ID [3]动作参数 |
| 0x0C | APP→设备 | 退货 | [0]订单号 [1]储物格号 |
| 0x0D | 设备→APP | 退货反馈 | [0]订单号 [1]格号 [2]有货 [3]门关 |
| 0x0E | APP→设备 | 检修 | 待定 |
| 0x10 | 双向 | 初始化查询 | [0]=0x01(已完成) |

**协议帧示例** (16进制):
```
APP发送订单:     AF 01 01 02 03 01 05 00...00 [CRC_L CRC_H]
设备确认接收:    BF 01 01 00...00 [CRC_L CRC_H]
设备投递成功:    BF 02 01 00 00 00...00 [CRC_L CRC_H]
初始化查询:      AF 10 01 00...00 [CRC_L CRC_H]
初始化完成响应:  BF 10 01 00...00 [CRC_L CRC_H]
```

### 2. UART2协议 (M109驱动卡通讯)

**帧格式** (20字节, 固定长度, 与UART3帧格式相同):
```
┌──────────┬──────────┬──────────────────┬──────────┐
│ 设备地址 │  功能码  │    数据域(Y/Z)   │  CRC16   │
│ 1B(0x01) │  1B      │    16B           │  2B(LE)  │
└──────────┴──────────┴──────────────────┴──────────┘
```

**M109指令码**:

| 指令码 | 名称 | 方向 | 数据域(Y/Z)说明 |
|--------|------|------|----------------|
| 0x03 | Motor Poll | 设备→M109 | Y: 无 / Z: [0]运行状态 [1]电机号 [2]结果 [3-4]峰值电流... |
| 0x05 | Motor Run | 设备→M109 | Y: [0]电机索引 [1]类型 [2]光幕模式 [3]延时... |
| 0x07 | Read Temp | 设备→M109 | Z: [0-1]温度×10 |
| 0x08 | Write DO | 设备→M109 | Y: [0]DO索引 [1]操作(ON/OFF) |

**M109电机类型** (Y2字段):

| 值 | 类型 | 说明 |
|----|------|------|
| 0x00 | 无反馈电磁铁 | - |
| 0x02 | 两线制电机 | - |
| 0x03 | 三线制电机 | - |
| 0x06 | 皮带货道 H→L→H | - |
| 0x07 | 皮带货道 L→H→L | 常用 |

**M109执行结果** (Z3字段):

| 值 | 结果 | 说明 |
|----|------|------|
| 0x00 | 成功 | 正常完成 |
| 0x01 | 过流 | 负载过重/卡货 |
| 0x02 | 欠流 | 电线断开/负载未安装 |
| 0x03 | 超时 | >7秒未检测到位 |
| 0x04 | 光幕失败 | 光幕自检失败 |
| 0x05 | 门未开 | 电磁铁门未开 |

### 3. UART1协议 (打包机PLC, Modbus RTU)

打包机PLC通过UART1使用标准 **Modbus RTU** 协议通信，由APP层 `app_proto` 模块封装：

**APP层 Modbus 通信函数** (封装了UART1发送+接收+解析):

| 函数 | 寄存器 | 功能 | 返回值 |
|------|--------|------|--------|
| `App_Proto_PackerSendStart()` | D29=1 | 发送启动命令 | 0=成功, 2=超时 |
| `App_Proto_PackerSendStop()` | D29=0 | 发送停止命令 | 0=成功, 2=超时 |
| `App_Proto_PackerReadTaskStatus(&val)` | 读D30 | 读取打印任务状态 | 0=成功, val:0/1 |
| `App_Proto_PackerTriggerBag()` | D31=1 | 触发出袋贴单 | 0=成功, 2=超时 |
| `App_Proto_PackerReadBagDone(&val)` | 读D32 | 读取出袋完成状态 | 0=成功, val:0/1 |
| `App_Proto_PackerClearBagDone()` | D32=0 | 清零出袋完成标志 | 0=成功, 2=超时 |
| `App_Proto_PackerReadFeedDone(&val)` | 读D36 | 读取投料完成状态 | 0=成功, val:0/1 |
| `App_Proto_PackerClearFeedDone()` | D36=0 | 清零投料完成标志 | 0=成功, 2=超时 |
| `App_Proto_PackerReadAlarm(&val)` | 读D35 | 读取报警码 | 0=成功, val:0~12 |
| `App_Proto_PackerReadAvoid(&val)` | 读D37 | 读取避让状态 | 0=成功, val:0~4 |
| `App_Proto_PackerWriteAvoid(val)` | 写D37 | 写入避让动作 | 0=成功, 2=超时 |

**内部实现**: 所有函数通过 `packer_modbus_read()` / `packer_modbus_write()` 两个静态通用函数实现，超时500ms。

**写命令回显验证** (v3.6新增): `packer_modbus_write()` 发送写命令后收到8字节回显时，调用 `BSP_Packer_VerifyWriteResp()` 对回显帧进行内容校验(memcmp + CRC16)，确保PLC确实正确回显了写命令，而非简单检查字节数。返回值: 0=成功(回显匹配), 1=回显不匹配, 2=超时。

### 4. BSP_Locker协议 (储物柜控制板, xy轴板↔储物板)

**帧格式** (20字节, 帧头0xCF):
```
┌──────────┬──────────┬──────────────────┬──────────┐
│ 帧头     │  指令码  │    数据域        │  CRC16   │
│ 0xCF     │  1B      │    16B           │  2B(LE)  │
└──────────┴──────────┴──────────────────┴──────────┘
```

> **重要**: 开锁(0x01)≠开门(0x03)。开锁打开电磁锁(800ms自断); 门控驱动大门电机。

#### 4.1 开锁指令 (0x01)

**发送 (xy轴板→储物板):**

| 字节 | 值 | 说明 |
|------|-----|------|
| [0] | 0xCF | 帧头 |
| [1] | 0x01 | 开锁指令码 |
| [2] | 0x01~0x10 | 锁编号 (1~16) |
| [3-17] | 0x00 | 保留 |
| [18-19] | CRC | CRC16校验 |

**返回 (储物板→xy轴板):**

| 字节 | 说明 |
|------|------|
| [0] | 0xCF 帧头 |
| [1] | 0x01 开锁指令码 |
| [2] | 锁编号 (1~16) |
| [3] | 锁状态 (0=关闭, 1=打开) |
| [4] | 货物状态 (1=有货, 0=无货) |
| [5-17] | 0x00 保留 |
| [18-19] | CRC16校验 |

#### 4.2 查询指令 (0x02)

**发送 (xy轴板→储物板):**

| 字节 | 值 | 说明 |
|------|-----|------|
| [0] | 0xCF | 帧头 |
| [1] | 0x02 | 查询指令码 |
| [2-17] | 0x00 | 无参数 |
| [18-19] | CRC | CRC16校验 |

**返回 (储物板→xy轴板):**

| 字节 | 说明 |
|------|------|
| [0] | 0xCF 帧头 |
| [1] | 0x02 查询指令码 |
| [2] | 锁状态高字节 (bit0~7 对应锁9~16, 预留) |
| [3] | 锁状态低字节 (bit0~5 对应锁1~6) |
| [4] | 货物状态高字节 (预留) |
| [5] | 货物状态低字节 (bit0~5 对应格1~6, 1=有货 0=无货) |
| [6-17] | 0x00 保留 |
| [18-19] | CRC16校验 |

**位掩码示例**: 锁状态`0x00 0x05`=锁1和锁3打开; 货物`0x00 0x3F`=格1~6全部有货

#### 4.3 门控指令 (0x03)

**发送 (xy轴板→储物板):**

| 字节 | 值 | 说明 |
|------|-----|------|
| [0] | 0xCF | 帧头 |
| [1] | 0x03 | 门控指令码 |
| [2] | 0x00/0x01 | 0x00=关门, 0x01=开门 |
| [3-17] | 0x00 | 保留 |
| [18-19] | CRC | CRC16校验 |

**返回 (储物板主动反馈):**

| 字节 | 说明 |
|------|------|
| [0] | 0xCF 帧头 |
| [1] | 0x03 门控指令码 |
| [2] | 执行结果 (0x00=开门完成, 0x01=关门完成) |
| [3] | 前限位状态 (IN13) |
| [4] | 后限位状态 (IN14) |
| [5-17] | 0x00 保留 |
| [18-19] | CRC16校验 |

**主动反馈机制**: 储物板检测到限位开关触发后**主动发送**门控反馈, 无需轮询:
- 开门完成: 前限位IN13=0时, 发送 `[2]=0x00`
- 关门完成: 后限位IN14=0时, 发送 `[2]=0x01`

**限位保护**: 触发限位后电机停转, 只能接收反方向指令:

| 限位状态 | 允许指令 | 禁止指令 |
|----------|----------|----------|
| 开门限位(IN13=0) | 关门(0x00) | 开门(0x01) |
| 关门限位(IN14=0) | 开门(0x01) | 关门(0x00) |

> 门控超时建议: 10秒

### 5. BSP_Packer协议 (打包机PLC, Modbus RTU)

**写命令帧** (8字节):
```
┌────────┬────────┬──────────────┬──────────┬──────────┐
│从机地址│功能码  │ 寄存器地址   │  数据值  │  CRC16   │
│0x01    │0x06    │ 2B(大端)     │ 2B(大端) │ 2B(小端) │
└────────┴────────┴──────────────┴──────────┴──────────┘
```

**寄存器地址映射**:

| PLC地址 | Modbus地址 | 功能 | R/W | 值域 |
|---------|-----------|------|-----|------|
| D29 | 0x101D | 设备启停 | R/W | 0=停止, 1=启动 |
| D30 | 0x101E | 任务有无 | R | 0=无, 1=有 |
| D31 | 0x101F | 触发出袋贴单 | W | 0=无, 1=触发 |
| D32 | 0x1020 | 出袋完成 | R/W | 0=未完成, 1=完成 |
| D33 | 0x1021 | 触发打包封口 | W | 0=无, 1=触发 |
| D34 | 0x1022 | 封口完成 | R/W | 0=未完成, 1=完成 |
| D35 | 0x1023 | 报警信息 | R | 0=正常, 1~12=各类报警 |
| D36 | 0x1024 | 投料完成 | R/W | 0=未完成, 1=完成 |
| D37 | 0x1025 | 避让动作 | R/W | 0~4各状态 |

**D37避让动作值含义**:

| 值 | 含义 | 写入方 |
|----|------|--------|
| 0 | 无动作 | — |
| 1 | 漏斗关闭+封口杆缩进 | 上位机写入 |
| 2 | 避让结束+漏斗打开+封口杆推出 | 上位机写入 |
| 3 | 缩进完成标志 | PLC写入 |
| 4 | 推出完成标志 | PLC写入 |

**封口杆避让流程** (Y轴移动保护):

所有Y轴运动前，自动执行 `packer_ensure_avoid()` 缩进逻辑:
1. 读取D37 → 若为1或3(封口杆已缩进)，直接放行
2. 若D37为0/2/4 → 写D37=1(请求缩进)，轮询等待PLC回写D37=3(缩进完成，最长10s)
3. Y轴运动执行
4. Y轴到达投料口(IN16)后，执行 `packer_restore_and_confirm()` 张开逻辑:
   - 写D37=2(请求推出), 轮询等待PLC回写D37=4(推出完成，仅接受D37=4，最长20s)
5. 投料完成后，Y轴反转回零前再次执行 `packer_ensure_avoid()` 缩进

**packer_ensure_avoid() 调用点** (v3.6统一):

| 调用位置 | 函数 | 场景说明 |
|----------|------|----------|
| check_xyz_positions() | Y轴回零前 | 出货前三轴位置检查 |
| check_xyz_for_seal() | Y轴回零前 | 封口前三轴位置检查 |
| process_order_send() 步骤I | 投料前 | 原内联D37代码已统一为此函数 |
| process_store_cmd() | Y轴正转/反转前 | 存放流程 |
| process_device_test() | Y轴自检/调试前 | 含常规Y正/反转及新增Y调试(0x0B) |

**Modbus帧示例**:
- 读D37: `01 03 10 25 00 01 91 01`
- 写D37=1(缩进): `01 06 10 25 00 01 5D 01`
- 写D37=2(恢复): `01 06 10 25 00 02 1D 00`

## 六、业务流程详解

### 1. 系统启动流程

```
1. MCU上电复位
   ↓
2. HAL初始化
   - GPIO / UART / Timer(TIM1/TIM4/TIM5/TIM8) / PWM 初始化
   - 中断向量表配置
   ↓
3. FreeRTOS初始化 (MX_FREERTOS_Init)
   - 创建信号量: xSem_UART1, xSem_UART2, xSem_UART3
   - 初始化LED + 创建LED软件定时器(10ms)
   - 创建队列: xQueue_Motion(32B×8), xQueue_Cmd(64B×8)
   - 创建7个任务
   - 创建事件组: xEvent_Motion
   - 初始化设备状态: App_State_Init() → STATE_IDLE
   ↓
4. tsk_Init 启动 (一次性初始化任务)
   - 状态设为 STATE_INITIALIZING
   - XZ并行自检: X正转→顶点(IN2)失能, Z正转→顶点(IN12)→停1s→反转→中点(IN13)失能
   - Y轴回零: 与打包机通讯确认封口杆缩回(D37避让) → Y反转→零点(IN5)失能 → 封口杆恢复张开 → D37清零
   - 成功: 设置EVT_INIT_OK, 发送初始化成功帧, 删除自身
   - 失败: 设置EVT_INIT_FAIL, 发送错误帧, 挂起
   ↓
5. tsk_Business 启动
   - 等待 EVT_INIT_OK 或 EVT_INIT_FAIL (超时30秒)
   - 成功: 设为 STATE_READY, 进入主循环等待指令
   - 失败: 设为 STATE_ERROR (允许0x10初始化查询和0x0B设备测试)
   ↓
6. 系统就绪
   - APP发送 0x10 初始化查询, 设备响应后开始工作
```

### 2. 订单出货流程 (0x01, 核心业务)

**业务模型**: APP每次发送一条0x01指令对应投递一件货物。投递成功后设备回复0x02,
APP再下发下一条0x01。所有0x01投递的货物属于同一个袋子，直到APP发送0x03打包封口。

```
APP发送: AF 01 [订单号 数量(保留) 货柜地址 货层 货道 ...] CRC
   ↓
[1] tsk_UART3Comm 解析帧, cmd_msg_t{src=3, func=0x01} → xQueue_Cmd
   ↓
[2] tsk_Business 取指令, 检查 App_State_IsReady()
   ├─ 非就绪: 丢弃(静默忽略)
   └─ 就绪: 设为 STATE_DELIVERING, 发送确认帧 BF 01 01 00...
   ↓
[3] XYZ轴位置校验 (check_xyz_positions):
   │
   ├─ X轴: 未在顶点(IN2) → 正转到顶点停止(失能)
   │
   ├─ Z轴: 未在中点(IN13):
   │    ├─ 已在顶点(IN12) → 反转到中点(IN13)停止(失能)
   │    └─ 其他位置 → 正转目标IN12, 副传感器IN13(途中触发则立刻停止)
   │         └─ 到达顶点(非中点) → 反转到中点(IN13)
   │
   ├─ Y轴移动前: 封口杆避让确认 packer_ensure_avoid() (v3.6新增)
   │    失败→上报ERR_PACKER_COMM_FAIL(0x21)→退出
   │
   ├─ Y轴: 未在零点(IN5) → 反转寻零到零点停止(失能)
   │
   ├─ Y轴到零点后: 封口杆张开确认 packer_restore_and_confirm() → 等待D37=4(推出完成) → 2秒后发送清零指令(D37=0)
   │
   └─ 任一轴10秒内未感应到传感器 → 上报ERR_POS_CHECK_FAIL(0x2B) → 退出
   ↓
[4] 检查 s_bag_ready 标志:
   │
   ├─ s_bag_ready == 0 (首件, 需要出袋):
   │    ├─ UART1发送打包机启动命令 (D29=1, 3次重试)
   │    │    失败→上报通信超时(断连)→退出
   │    │    成功→5秒延时(等待打包机就绪)
   │    ├─ UART1读取打印任务状态 (D30)
   │    │    无打印内容(D30=0)→上报货物异常→退出
   │    ├─ UART1触发出袋贴单 (D31=1)
   │    │    通信失败→上报超时→退出
   │    ├─ UART1轮询出袋完成 (D32, 每200ms, 3秒超时)
   │    │    D32=1→清零D32→标记 s_bag_ready=1
   │    │    超时→上报操作超时→退出
   │    └─ 进入投递步骤
   │
   └─ s_bag_ready == 1 (同一袋子的后续件, 袋子已出):
        └─ 直接进入投递步骤
   ↓
[5] 单件投递流程 (步骤A~L):
   │
   ├─ 步骤A: X轴正转到顶点(IN2)停止(失能) + UART2发送M109电机启动指令 (0x05)
   │    计算motor_idx = (layer-1)*10 + (slot-1)
   │    电机类型: H→L→H (0x06), 光幕模式: OFF (0x00)
   │
   ├─ 步骤B: 等待3秒 (M109电机运转)   │
   ├─ 步骤C: UART2发送M109查询指令(0x03), 最多10次轮询
   │    等待 run_state == M109_RUN_DONE (0x02)
   │
   ├─ 步骤D: 检查M109执行结果
   │    失败→发送投递失败帧(0x02)→退出 (s_bag_ready保持1, 袋已出)
   │
   ├─ 步骤D2: 货柜号判断 (cabinet_addr ≥ 4 时执行)
   │    ├─ 启动传送带2 (TIM5 CH2, NMOS2缓启动 100%/30ms)
   │    ├─ 等待IN17按下(低电平) — 转盘/货物到位 (30秒超时)
   │    │    超时→停止传送带2→上报ERR_FEED_TIMEOUT(0x26)→退出
   │    └─ IN17触发→停止传送带2→继续步骤E
   │    (货柜号1~3时跳过此步, 直接进入步骤E)
   │
   ├─ 步骤E: TIM5 CH1 NMOS缓启动 (100%占空比, 30ms步进)
   │
   ├─ 步骤F: 等待IN1按下(低电平) — 货物到达入口 (30秒超时)
   │
   ├─ 步骤G: IN1拉低后延时2秒, 停止NMOS传送带电机
   │
   ├─ 步骤H: X轴反转到中点(IN3)停止(失能)
   │
   ├─ 步骤I: 封口杆避让确认 — 调用 packer_ensure_avoid() (v3.6统一)
   │    ├─ D37=0/2/4(伸开状态)→发送缩回指令(D37=1)→轮询等待D37=3(10秒超时)
   │    └─ D37=1/3(已缩回)→直接继续
   │
   ├─ 步骤J: Y轴正转到投料入口IN16停止(使能保持)
   │
   ├─ 步骤K: 封口杆张开确认
   │    ├─ 发送张开指令(D37=2)
   │    ├─ 等待2秒
   │    └─ 查询D37, 确认D37=4(张开完成) 或 D37=2(指令已发出)
   │
   └─ 步骤L: 投料电机正转(TIM1互补PWM, IR2104 NMOS) + D36轮询 + 缩回 + Y回零 + 张开 + 清除D37
        ├─ 启动投料电机正转(TIM1互补输出, SD使能)
        ├─ 每1秒轮询D36, 10秒总超时
        ├─ D36=1→立即停止投料电机→清零D36(检查返回指令，未收到则重试，最多3次)
        ├─ 发送封口杆缩回指令(D37=1)
        ├─ 2秒后开始轮询D37=3(缩回完成), 20秒超时
        ├─ 缩回确认→Y轴反转回零IN5(失能)→2秒后发送封口杆张开指令(D37=2)→等待PLC回写D37=4(推出完成)→2秒后清除D37=0
        └─ 投料超时→上报操作超时→退出 (s_bag_ready保持1, 袋已出)
   ↓
[6] UART3发送投递成功帧 (BF 02 01 00 00 ...)
   ↓
[7] 设为 STATE_READY (s_bag_ready保持为1, 等待下一条0x01或0x03)
```

**多件货物一袋的完整时序**:
```
APP: 0x01 (第1件)  →  设备: PLC启动→出袋→投递→0x02成功
APP: 0x01 (第2件)  →  设备: 直接投递(跳过出袋)→0x02成功
APP: 0x01 (第3件)  →  设备: 直接投递(跳过出袋)→0x02成功
APP: 0x03 (打包封口) →  设备: 打包→封口→0x04完成→s_bag_ready=0
APP: 0x01 (新袋第1件) → 设备: PLC启动→出袋→投递→0x02成功
...
```

**投递失败时 s_bag_ready 不重置设计说明**:
- 袋子一旦出完(s_bag_ready=1), 投递步骤(A~L)失败**不会**将 s_bag_ready 重置为 0
- 这样下次 APP 重发 0x01 时直接重试投递, 跳过出袋流程, 避免旧袋已有货物却再出新袋的问题
- s_bag_ready 仅在两种情况下为 0:
  - 系统上电初始化(静态变量默认值)
  - APP 发送 0x03 打包封口完成后(当前袋子已封口, 需要出新袋)
- 失败重试场景示例:
  ```
  APP: 0x01 (第1件)  →  成功 → s_bag_ready=1
  APP: 0x01 (第2件)  →  失败(M109异常) → s_bag_ready保持1
  APP: 0x01 (重试第2件) → 直接投递(跳过出袋) → 成功
  APP: 0x03 (打包封口)  → s_bag_ready=0
  ```

### 3. 打包封口流程 (0x03)

```
APP发送: AF 03 [订单号 0x01 ...] CRC
   ↓
[1] 发送打包封口响应 (0x03)
   ↓
[2] XYZ轴位置校验 (check_xyz_for_seal):
   │
   ├─ X轴: 未在中点(IN3):
   │    ├─ 已在顶点(IN2) → 反转到中点(IN3)停止(失能)
   │    └─ 其他位置 → 正转目标IN2, 副传感器IN3(途中触发则立刻停止)
   │         └─ 到达顶点(非中点) → 反转到中点(IN3)
   │
   ├─ Z轴: 未在顶点(IN12) → 正转到顶点(IN12)停止(失能)
   │
   ├─ Y轴: 已在零点(IN5) → 无需移动, 无需封口杆避让, 直接失能
   │        未在零点 → 封口杆避让确认 packer_ensure_avoid() → 反转寻零到零点停止(失能)
   │
   └─ 任一轴10秒内未感应到传感器 → 上报ERR_POS_CHECK_FAIL(0x2B) → 退出
   ↓
[3] 封口杆张开确认:
    查询D37 → 若D37=4(已张开)或D37=0(无动作) → 跳过, 直接进入下一步
    否则 → 发送压合杆张开指令(D37=2, packer_restore_and_confirm)
         → 等待PLC确认张开完成(D37=4)
         → 2秒后发送封口杆清零(D37=0)
   ↓
[4] UART1发送封口完成清零(D34=0), 然后发送打包封口触发命令 (D33=1)
    轮询封口完成 (D34, 每500ms, 30秒超时)
    D34=1 → 启动Motor4反转5秒后停止 → 清零D34
    超时 → 上报ERR_PACKER_COMM_FAIL
    发送打包机停止指令 (D29=0)
   ↓
[5] 发送封口完成反馈 (0x04, data[0]=0xAA)
   ↓
[6] 重置 s_bag_ready=0, 设为 STATE_READY
```

### 4. 储物格存放流程 (0x05)

```
APP发送: AF 05 [打包编号 储物格号 ...] CRC
   ↓
[1] 发送存放响应 (0x05)
[2] 封口杆避让确认 + UART2发送开门帧(BSP_Locker 0xCF/0x03, action=0x01开门)
    Y轴正转→储物格格号对应传感器位置停止(保持)
    储物格号1~6映射到取货口传感器IN11~IN6
[3] 等待储物板主动反馈开门完成(0xCF/0x03, result=0x00 + data[2]=0x01开门传感器确认)
    Y轴保持使能等待, 30秒总超时

    
    超时→Y轴返回零点(IN5)，回零成功则失能→报错
[4] X轴反转→X零点(IN4)
[5] Z轴反转→Z零点(IN14) → 延时1秒(CFG_Z_STOP_DELAY_MS)
[6] X轴正转→X顶点(IN2)，Z轴正转→Z中点(IN13)
[7] 发送关门帧(BSP_Locker 0xCF/0x03, action=0x00关门)
    10秒内等待关门确认返回帧(result=0x01 + data[1]=0x01关门传感器确认)
    未收到→重发, 最多3次重试
    3次均失败→上报门未关闭报警(ERR_DOOR_FAIL)→终止
    成功→继续
[8] 封口杆避让确认(未缩回则发送缩回指令, 等待缩回完成) + Y轴回零（失能）
    + 发送封口杆张开指令(D37=2), 等待PLC回写D37=4(推出完成), 2秒后清除D37=0
[10] 串口2发送储物格状态查询指令(0xCF/0x02)
[11] 更新储物格占用状态, 发送存放反馈(0x06)
[12] 设为 STATE_READY
  失败: 紧急停止所有电机, 发送失败反馈
```

### 6. 设备状态查询流程 (0x09)

```
APP发送: AF 09 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
   ↓
[1] UART2发送储物板查询指令 (BSP_Locker 0xCF/0x02)
[2] 等待储物板查询响应 (3秒超时)

    超时: 使用本地缓存值
[3] 组装响应帧 BF 09 [lock_high lock_low occupy_high occupy_low 00...] CRC
    发送给APP
```

**示例:**
- APP发送: `AF 09 01 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 77 13`
- 设备响应: `BF 09 00 15 00 0A 00 00 00 00 00 00 00 00 00 00 00 00 7F 43`

  - `0x0A` = 0000 1010 → 格2、4有货

**Y轴逐级升降指令详细说明** (v3.8重构):

> Y轴内部维护一个持久化位置状态变量 `s_y_step_pos`（0~6），跨指令调用保持。
> 0=零点(IN5)，1=IN6，2=IN7，3=IN8，4=IN9，5=IN10，6=IN11。

**传感器级别对照表**:

| 级别(s_y_step_pos) | 传感器 | GPIO | 说明 |
|---------------------|--------|------|------|
| 0 | IN5 (CFG_SENSOR_Y_ZERO) | PB14 | 零点(Y轴失能位) |
| 1 | IN6 (CFG_SENSOR_PICKUP_1) | PB1 | 第1停止点 |
| 2 | IN7 (CFG_SENSOR_PICKUP_2) | PB0 | 第2停止点 |
| 3 | IN8 (CFG_SENSOR_PICKUP_3) | PC5 | 第3停止点 |
| 4 | IN9 (CFG_SENSOR_PICKUP_4) | PC4 | 第4停止点 |
| 5 | IN10 (CFG_SENSOR_PICKUP_5) | PA7 | 第5停止点 |
| 6 | IN11 (CFG_SENSOR_PICKUP_6) | PA6 | 顶端(上升上限) |

**指令0x0B — Y轴逐级上升**:
```
APP发送: AF 0B [type=0x02 00 00 action=0x0B 00 ...] CRC
   ↓
[0] 零点同步校准: 若 s_y_step_pos≠0 且 IN5(零点传感器)触发
    → 说明其他业务已将Y轴回零, 自动修正 s_y_step_pos=0
   ↓
[1] 检查当前位置 s_y_step_pos:
    ├─ pos >= 6 (已在IN11顶端) → 静默忽略，返回TEST_RESULT_OK
    └─ pos < 6 → 继续
   ↓
[2] 封口杆避让确认 packer_ensure_avoid()
    失败 → 返回 TEST_RESULT_TIMEOUT
   ↓
[3] Y轴正转(MOTION_CMD_Y_FORWARD)到目标传感器 y_step_sensors[pos]
    目标传感器: pos=0→IN6, 1→IN7, ..., 5→IN11
    到位自动停止，保持使能(不调用BSP_Stepper_Disable)
    运动失败/超时 → 返回 TEST_RESULT_STUCK / TEST_RESULT_TIMEOUT
   ↓
[4] 成功: s_y_step_pos++ → 返回 TEST_RESULT_OK
    APP可重复发送0x0B，每次上升一级，最多上升至IN11
```

**指令0x0C — Y轴逐级下降**:
```
APP发送: AF 0B [type=0x02 00 00 action=0x0C 00 ...] CRC
   ↓
[0] 零点同步校准: 若 s_y_step_pos≠0 且 IN5(零点传感器)触发
    → 自动修正 s_y_step_pos=0
   ↓
[1] 检查当前位置 s_y_step_pos:
    ├─ pos <= 0 (已在零点) → 静默忽略，返回 TEST_RESULT_OK
    └─ pos > 0 → 继续
   ↓
[2] 封口杆避让确认 packer_ensure_avoid()
    失败 → 返回 TEST_RESULT_TIMEOUT
   ↓
[3] Y轴反转(MOTION_CMD_Y_REVERSE)到目标传感器 y_step_down_sensors[pos-1]
    目标传感器: pos=6→IN10, 5→IN9, ..., 2→IN6, 1→零点(IN5)
    到位自动停止
    运动失败/超时 → 返回 TEST_RESULT_STUCK / TEST_RESULT_TIMEOUT
   ↓
[4] 成功: s_y_step_pos--
    ├─ pos 降至 0 (到达零点) → 调用 BSP_Stepper_Disable(Y轴失能)
    └─ pos > 0 → 保持使能，等待下一条指令
    返回 TEST_RESULT_OK
```

**指令0x0D — Y轴立即停止**:
```
APP发送: AF 0B [type=0x02 00 00 action=0x0D 00 ...] CRC
   ↓
[1] 发送 MOTION_CMD_STOP_ALL → 运动任务立即停止所有电机
[2] 保持Y轴使能(不调用BSP_Stepper_Disable)，维持保持力矩防止滑落
[3] s_y_step_pos 不变(保留最后已到达的级别，下次升降从该级出发)
[4] 返回 TEST_RESULT_OK
```

**完整使用示例**:
```
初始状态: Y轴在零点 (s_y_step_pos=0)

APP → 0x0B(action=0x0B) → 设备: Y轴 零点→IN6 停止使能 → s_y_step_pos=1 → 返回OK
APP → 0x0B(action=0x0B) → 设备: Y轴 IN6→IN7 停止使能  → s_y_step_pos=2 → 返回OK
...（重复，每次上升一级）
APP → 0x0B(action=0x0B) → 设备: Y轴 IN10→IN11 停止使能 → s_y_step_pos=6 → 返回OK
APP → 0x0B(action=0x0B) → 设备: 已在顶端，静默忽略    → s_y_step_pos=6 → 返回OK

APP → 0x0B(action=0x0C) → 设备: Y轴 IN11→IN10 停止使能 → s_y_step_pos=5 → 返回OK
...（重复，每次下降一级）
APP → 0x0B(action=0x0C) → 设备: Y轴 IN6→零点(IN5) 失能 → s_y_step_pos=0 → 返回OK
APP → 0x0B(action=0x0C) → 设备: 已在零点，静默忽略    → s_y_step_pos=0 → 返回OK

APP → 0x0B(action=0x0D) → 设备: 立即停止+保持使能+pos不变 → 返回OK
```

**注意**: 停止指令(0x0D)执行后`s_y_step_pos`保留不变，后续可继续使用上升/下降指令。
Y轴保持使能均维持保持力矩，防止滑落。若需完全释放保持力，需通过其他方式处理。

### 8. 寻零流程

```
初始化任务流程:

第1步: BSP层初始化
  - GPIO/UART/Stepper/NMOS/Timer/Motor4 依次初始化

第2步: X+Z并行自检 (TIM4 ARR共用, 同频)
  - X轴正转(CW) → 顶点IN2 → 失能
  - Z轴正转(CW) → 顶点IN12 → 停1秒 → 反转 → 中点IN13 → 失能
  - 缓启动: 1100Hz线性加速到CFG_SEEKZERO_FREQ
  - 超时30秒

第3步: Y轴回零 (需封口杆避让)
  - 等待500ms确保UART1任务就绪
  - 读取打包机D37寄存器:
    · D37=1或3 → 封口杆已缩回, 直接进行Y轴回零
    · 其他值 → 写D37=1请求缩回, 轮询等待D37=3(10s超时)
  - Y轴反转回零(BSP_Stepper_SeekZero) → 零点IN5 → 失能
  - 封口杆恢复: 写D37=2 → 等待D37=4 → 写D37=0清零

全部完成:
  - 设置EVT_INIT_OK事件位
  - 发送初始化成功帧到APP (UART3)
  - tsk_Init自删除

任一步骤失败:
  - 设置EVT_INIT_FAIL事件位
  - 发送ERR_SEEKZERO_FAIL错误帧
  - tsk_Init挂起(需断电重启)
```

### 9. 异常处理与指令并发

#### 9.1 业务流程互斥保护

- **重业务指令互斥**: 订单下发(0x01)、打包封口(0x03)、储物格存放(0x05)三个流程涉及运动轴/传送带/打包机，必须按顺序执行，同一时刻只能运行一个。非就绪状态下收到这三个指令或设备测试(0x0B)，返回 **ERR_RES_OCCUPIED(0x28)**
- **调试模式仅限初始化后立即使用**: 设备测试(0x0B)只允许在初始化完成后、尚未执行任何业务流程时进入。一旦执行过订单下发(0x01)、打包封口(0x03)或储物格存放(0x05)中的任何一个，调试模式将被永久锁定，后续收到设备测试指令返回 **ERR_TEST_LOCKED(0x2C)**（需断电重启才能重新进入调试模式）
- **自检失败时允许测试**: 当设备处于 STATE_ERROR（自检失败）状态时，除初始化查询(0x10)外，设备测试指令(0x0B)也可正常接收和执行，便于排查硬件故障
- **轻量指令可并发**: 取货开锁(0x07)、退货(0x0C)、状态查询(0x09)、初始化查询(0x10)可在重业务流程执行期间同时处理，无需等待当前流程结束

#### 9.2 轻量指令并发机制

**实现方式**: 在重业务流程的长耗时轮询循环中，周期性调用 `poll_lightweight_cmds()` 从指令队列中取出轻量指令立即处理。

**轮询插入点**:

| 重业务流程 | 插入位置 | UART2状态 | 可处理 |
|-----------|---------|-----------|--------|
| 出货(0x01) | D32出袋轮询(500ms间隔) | 空闲 | 取货/退货/查询 |
| 出货(0x01) | IN1传感器检测(200ms间隔) | 空闲 | 取货/退货/查询 |
| 出货(0x01) | D36投料轮询(1s间隔) | 空闲 | 取货/退货/查询 |
| 出货(0x01) | D37缩回轮询(500ms间隔) | 空闲 | 取货/退货/查询 |
| 打包(0x03) | D34封口轮询(500ms间隔) | 空闲 | 取货/退货/查询 |
| 存放(0x05) | 开门等待(1s间隔) | 占用 | 仅查询(缓存值) |
| 存放(0x05) | Z轴停留延时 | 空闲 | 取货/退货/查询 |

**UART2安全机制**:
- `uart2_safe=1`: UART2空闲，可执行取货/退货(需UART2与储物板通信)和状态查询(需UART2查询储物板)
- `uart2_safe=0`: UART2占用中(门控操作等)，取货/退货返回ERR_RES_OCCUPIED，状态查询使用本地缓存值回复

**状态保护**: 轻量指令执行前保存当前设备状态(如STATE_DELIVERING)，执行后自动恢复，不影响主业务流程状态机

#### 9.3 不可调度时段

- **M109电机轮询期间**(出货流程): M109查询使用UART2且持续10~20秒，期间轻量指令在队列中等待，M109完成后在下一个轮询点处理
- **闸门开关期间**(存放流程): UART2用于门控通信，取货/退货暂不可执行

#### 9.4 通用异常处理

- 运动失败: 紧急停止所有电机 (MOTION_CMD_STOP_ALL)
- LED模式自动切换至 LED_MODE_ERROR (200ms快闪5次)
- 业务流程正常完成或异常退出后，状态自动回到STATE_READY，可接受新指令

### 10. 错误上报指令 (0x0A, 通讯板→APP)

**功能**: 设备运行过程中遇到异常时，统一通过 **0x0A** 帧主动上报错误码给APP。
所有错误均通过此指令码上报，每个错误码唯一，互不冲突。

**帧格式**: `BF 0A [data 16B] CRC16`

| 字节偏移 | 字段名 | 字节数 | 取值范围 | 说明 |
|----------|--------|--------|----------|------|
| 0 | 错误码 | 1 | 0x01~0x2D | 参考下方统一错误码表 |
| 1~15 | 预留字段 | 15 | 0x00 | 补全16字节 |

**示例**: `BF 0A 21 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC_L CRC_H`
→ 上报错误码 0x21 (打包机PLC通信失败)

#### 10.1 系统/硬件类错误码 (0x01~0x07)

| 错误码 | 宏名称 | 含义 | 触发场景 |

| 0x05 | ERR_DOOR_FAIL | 储物格门/锁失败 | 开锁/关门通用失败 |
| 0x06 | ERR_STATE_BUSY | 设备忙拒绝指令 | 非就绪状态收到不可排队的指令 |

## 八、CRC16计算

```c
/**
 * CRC16-Modbus 计算函数
 * 多项式: 0x8005 (反转后 0xA001)
 * 初始值: 0xFFFF
 * 字节序: 低字节在前
 */
uint16_t BSP_CRC16_Calc(const uint8_t *pData, uint16_t len)
{
    uint16_t crc = 0xFFFF;
    for (uint16_t i = 0; i < len; i++) {
        crc ^= pData[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x0001)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc >>= 1;
        }
    }
    return crc;
}

/* 追加CRC到帧尾 */
BSP_CRC16_Append(frame, 18);  // frame[18]=CRC_L, frame[19]=CRC_H

/* 校验完整帧 */
if (BSP_CRC16_Check(rx_frame, 20) == 1) { /* CRC正确 */ }
```

### 2. Python上位机示例

```python
import serial
import struct

def crc16_modbus(data):
    """CRC16-Modbus校验"""
    crc = 0xFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            if crc & 0x0001:
                crc = (crc >> 1) ^ 0xA001
            else:
                crc >>= 1
    return crc

def build_frame(addr, func, data_bytes=None):
    """构建20字节协议帧"""
    frame = bytearray(20)
    frame[0] = addr
    frame[1] = func
    if data_bytes:
        for i, b in enumerate(data_bytes[:16]):
            frame[2 + i] = b
    crc = crc16_modbus(frame[:18])
    frame[18] = crc & 0xFF
    frame[19] = (crc >> 8) & 0xFF
    return bytes(frame)

def parse_frame(data):
    """解析20字节协议帧"""
    if len(data) != 20:
        return None
    crc_calc = crc16_modbus(data[:18])
    crc_recv = data[18] | (data[19] << 8)
    if crc_calc != crc_recv:
        return None
    return {'addr': data[0], 'func': data[1], 'data': data[2:18]}

# 使用示例 (通过UART3通讯)
if __name__ == '__main__':
    uart3 = serial.Serial('COM3', 115200, timeout=2)

    # 查询初始化状态
    frame = build_frame(0xAF, 0x10, [0x01])
    uart3.write(frame)
    resp = uart3.read(20)
    if resp:
        parsed = parse_frame(resp)
        if parsed and parsed['addr'] == 0xBF:
            print(f"初始化状态: {parsed['data'][0]}")

    # 下单出货
    frame = build_frame(0xAF, 0x01, [0x01, 0x02, 0x03, 0x01, 0x05])
    uart3.write(frame)
    resp = uart3.read(20)
    if resp:
        parsed = parse_frame(resp)
        if parsed:
            print(f"订单响应: func=0x{parsed['func']:02X}")

    # 查询设备状态
    frame = build_frame(0xAF, 0x09)
    uart3.write(frame)
    resp = uart3.read(20)
    if resp:
        parsed = parse_frame(resp)
        if parsed:
            lock_status = parsed['data'][0] | (parsed['data'][1] << 8)
            occupy_status = parsed['data'][2] | (parsed['data'][3] << 8)
            print(f"锁状态: 0x{lock_status:04X}, 占用: 0x{occupy_status:04X}")

    uart3.close()
```

## 十一、故障排查指南

### 1. 轴寻零失败

| 症状 | 可能原因 | 排查步骤 |
|------|----------|---------|
| 某轴一直转 | 传感器失效 | 万用表测传感器电压; 调用BSP_GPIO_GetInState(IN)检查 |

### 2. 通讯故障

| 症状 | 可能原因 | 排查步骤 |
|------|----------|---------|

## 十三、注意事项

1. **串口分配**
   - UART1=打包模块, UART2=M109/储物板, UART3=APP上位机
   - 所有帧CRC16校验, APP协议地址0xAF/0xBF, M109协议地址0x01

2. **电源管理**
   - 各轴电机需独立供电, 避免干扰MCU
   - NMOS启动时电流冲击大, IR2104 SD引脚确保先设PWM再使能

3. **运动控制**
   - 寻零频率固定4000Hz, X/Z运行频率默认8000Hz, Y轴运行频率默认10000Hz
   - 寻零超时X/Z轴10秒, Y轴25秒; 初始化寻零30秒
   - 使能EN低有效: 运动前Enable, 到位后Disable

4. **并发控制**
   - 业务任务同一时刻只执行一个流程 (通过状态机互斥)
   - 非就绪状态收到订单指令直接**静默丢弃**
   - 打包/存储繁忙时回复忙碌错误码

5. **故障恢复**
   - M109通讯失败: 最多查询10次
   - 货柜通讯: 超时2秒, 重试3次
   - 打包机PLC通讯: 启动命令3次重试, 出袋完成3秒超时, 投料完成10秒超时
   - 运动失败: 紧急停止所有电机 (STOP_ALL)
   - 错误状态仅接受0x10初始化查询
